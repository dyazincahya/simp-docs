Installation Guide

Getting started with your new documentation template is straightforward. Follow these steps to set up your project and begin adding your content.

1. Download the Template

First, ensure you have downloaded the complete template package. You should have a folder structure similar to this:


my-docs/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── script.js
└── content/
    ├── documentation-intro.html
    ├── installation.html
    └── (other content files)

Unzip the downloaded package to your desired location on your local machine.

2. Open with a Code Editor

Open the `my-docs` folder (or whatever you named it) in your favorite code editor (e.g., VS Code, Sublime Text, Atom). This will allow you to easily navigate and edit the project files.

3. View in Browser

You can directly open `index.html` in your web browser. Since this is a static HTML template, it doesn't require a web server (like Apache or Nginx) to run locally, though using a local development server (e.g., with VS Code's Live Server extension) is recommended for features like live reloading.

Important Note on Local File Paths

When opening `index.html` directly using a `file://` URL, some browser security restrictions (e.g., related to AJAX requests) might prevent dynamic content loading from working correctly. For the best experience, especially when loading content via AJAX, it's highly recommended to use a simple local web server.

4. Customization (Optional)

You can customize the look and feel of the documentation by editing the `css/style.css` file. For more advanced styling, you might consider using the original Sass files if provided in the full template package (refer to the Sass documentation section if available).

How to Create a New Documentation Page

Adding new documentation sections or categories is simple, thanks to the modular structure of this template. Each main documentation section resides in its own HTML file.

1. Create a New Content File

Inside the `content/` directory, create a new HTML file for your new documentation section. Use a descriptive filename (e.g., `my-new-feature.html`, `api-endpoints.html`).

Important: This file should **only** contain the content of your `<section>`. Do not include `<html>`, `<head>`, or `<body>` tags in these content files.


<!-- content/my-new-feature.html -->
<section id="my-new-feature" class="mb-5 doc-section">
    <h2 class="h3 fw-bold border-bottom pb-2 mb-4">My Awesome New Feature</h2>
    <p>This is the detailed documentation for my awesome new feature.</p>
    
    <h3>Usage Example</h3>
    <pre><code class="language-js">
function initializeFeature() {
    // Your code here
    console.log("Feature initialized!");
}
    </code></pre>
    
    <p>Remember to include any necessary code snippets or images.</p>
</section>

Make sure the `id` attribute of the `<section>` tag is unique and matches the hash (`#`) you'll use in your sidebar link.

2. Add a Link to the Sidebar (`index.html`)

Now, you need to link your new content file from the sidebar navigation. Open `index.html` and locate the `<nav id="sidebar">` section.

Find the appropriate category (`<li class="nav-item my-1">`) where you want to add your new link. If you need a new top-level category, you can create a new `<li>` element.


<!-- ... inside your sidebar <ul class="sidebar-nav"> ... -->
    <!-- Example: Adding under "Getting Started" -->
    <div class="collapse show" id="gettingStartedCollapse">
        <ul class="nav flex-column ms-3">
            <li class="nav-item"><a class="nav-link nav-link-custom" data-content-file="installation.html" href="#installation"><span class="nav-link-text">Installation</span></a></li>
            <li class="nav-item"><a class="nav-link nav-link-custom" data-content-file="my-new-feature.html" href="#my-new-feature"><span class="nav-link-text">My New Feature</span></a></li> <!-- YOUR NEW LINK -->
        </ul>
    </div>

    <!-- Example: Adding as a new top-level category -->
    <li class="nav-item my-1">
        <a class="nav-link text-dark fw-bold nav-link-custom"
            data-content-file="api-endpoints.html" href="#api-endpoints"><span class="nav-link-text">API Endpoints</span></a>
    </li>
<!-- ... rest of your sidebar ... -->

Key attributes for your new link:

  • class="nav-link nav-link-custom": Essential for the JavaScript to handle clicks and dynamic loading.
  • data-content-file="your-new-file.html": This attribute tells the JavaScript which content file to load.
  • href="#your-section-id": This should match the `id` of the `<section>` tag inside your new content file.
  • <span class="nav-link-text">Your Page Title</span>: Wrap your link text in this `<span>` for proper search highlighting.

3. Test Your New Page

Save both `index.html` and your new content file. Refresh your browser. Your new link should appear in the sidebar. Click on it to ensure your content loads correctly and that syntax highlighting (if applicable) is working as expected.

4. Add to Initial Load (Optional)

If you want your new page to be visible by default when the documentation first loads (without needing to click a sidebar link), you can add it to the initial load sequence in `js/script.js`:


// Inside $(document).ready(function() { ... }); in js/script.js

// Add your new page here if you want it to load initially
loadContentAndScroll('#my-new-feature', 'my-new-feature.html', true, false);

Only do this if you intend for multiple sections to appear on the initial page load.
