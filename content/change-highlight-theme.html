Changing Highlight.js Theme

The code blocks in this documentation use Highlight.js for syntax highlighting. Highlight.js comes with a variety of beautiful themes that you can easily switch between to match your documentation's aesthetic.

1. Choose Your Desired Theme

Before making any changes, it's highly recommended to preview the available themes. Visit the official Highlight.js demo page:

Browse Highlight.js Themes

On the demo page, you can select different themes from the sidebar to see how they look with various code languages. Note down the exact filename of the theme's CSS (e.g., `atom-one-dark.min.css`, `github.min.css`, `dracula.min.css`).

2. Locate the Theme Link in `index.html`

Open your project's main `index.html` file in your code editor. In the `<head>` section, you will find the link to the Highlight.js theme stylesheet. It will look something like this:


<!-- Highlight.js CSS (pilih tema favorit Anda, contoh: atom-one-dark) -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/atom-one-dark.min.css">

3. Replace the Theme Filename

Change the filename in the `href` attribute to the new theme you selected. Ensure you maintain the correct path (`https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/`) and just swap out the theme name.

Example: Changing to "Dracula" Theme


<!-- Highlight.js CSS (tema Dracula) -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/dracula.min.css">

Example: Changing to "GitHub" Theme


<!-- Highlight.js CSS (tema GitHub) -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/github.min.css">

Always use the `.min.css` version for production to ensure smaller file sizes.

4. Save and Refresh Your Browser

Save the changes to your `index.html` file. Then, open or refresh your documentation in the web browser. It's recommended to perform a **hard refresh** (Ctrl + F5 on Windows/Linux, Cmd + Shift + R on Mac) to ensure your browser loads the new stylesheet and not a cached version.

Tip: Experimentation

Feel free to try out several themes until you find one that perfectly complements the overall design of your documentation. You can also mix a light theme for code with a dark documentation theme, or vice versa, to create visual contrast.
