HTML Structure

Understanding the core HTML structure of this documentation template will help you navigate the project and make modifications confidently. The project is primarily composed of two types of HTML files: the main `index.html` and individual content files located in the `content/` directory.

1. Main `index.html` File

The `index.html` file serves as the single entry point for your documentation. It includes the overall layout, global stylesheets, JavaScript libraries, and the dynamic content area.

Key Sections of `index.html`:

  • <head>: Contains metadata, title, external CSS links (Bootstrap, Font Awesome, Highlight.js, custom styles), and any other head-level configurations.
  • <body>:
    • <header>: The top navigation bar, usually containing the brand logo, search input, and global navigation links.
    • Main Content Wrapper: Typically a `<div class="container-fluid">` or `<div class="container">` that wraps the sidebar and the main content area.
      • <nav id="sidebar">: The left-hand navigation sidebar, which dynamically links to your content files.
      • <main id="content">: The primary area where your documentation content is loaded dynamically via AJAX.
    • <footer> (Optional): A common area for copyright information or global links at the bottom of the page.
    • JavaScript Files: All JavaScript libraries (jQuery, Bootstrap, Highlight.js) and your custom `script.js` are loaded just before the closing `</body>` tag for optimal performance.

Example `index.html` (Simplified Structure):


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Documentation</title>
    <!-- Bootstrap CSS -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
    <!-- Font Awesome -->
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.2/css/all.min.css">
    <!-- Highlight.js CSS -->
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/atom-one-dark.min.css">
    <!-- Custom CSS -->
    <link rel="stylesheet" href="css/style.css">
</head>
<body>
    <!-- Header (Navbar with Search) -->
    <header class="navbar navbar-expand-lg bg-dark sticky-top">
        <!-- ... header content ... -->
    </header>

    <div class="container-fluid">
        <div class="row">
            <!-- Sidebar Navigation -->
            <nav id="sidebar" class="col-lg-3 col-xl-2">
                <!-- ... sidebar links ... -->
            </nav>

            <!-- Main Content Area -->
            <main id="content" class="col-lg-9 col-xl-10 py-4">
                <!-- Content is dynamically loaded here -->
            </main>
        </div>
    </div>

    <!-- Optional Footer -->
    <!-- <footer class="footer mt-auto py-3 bg-light">...</footer> -->

    <!-- jQuery -->
    <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script>
    <!-- Bootstrap JS -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
    <!-- Highlight.js -->
    <script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/highlight.min.js"></script>
    <script>hljs.highlightAll();</script>
    <!-- Custom Script -->
    <script src="js/script.js"></script>
</body>
</html>

2. Individual Content Files (`content/` directory)

Each distinct documentation page or section is stored as a separate HTML file within the `content/` directory. These files are loaded into the `<main id="content">` area of `index.html` when their corresponding sidebar link is clicked.

Structure of a Content File:

Crucially, these content files should **only** contain the HTML relevant to their section. They must **not** include full HTML document tags like `<!DOCTYPE html>`, `<html>`, `<head>`, or `<body>`. jQuery's `.load()` method is designed to insert only the content of the specified file directly into the target element.

Example `content/your-page.html` File:


<!-- content/your-page.html -->
<section id="your-page-id" class="mb-5 doc-section">
    <h2 class="h3 fw-bold border-bottom pb-2 mb-4">Your Page Title</h2>
    <p>This is the content of your documentation page.</p>

    <h3>Code Example:</h3>
    <pre><code class="language-js">
// JavaScript example
console.log("Hello from your page!");
    </code></pre>

    <h4>Sub-section</h4>
    <p>More details about this section.</p>
</section>

Remember to assign a unique `id` to your `<section>` tag, as this `id` will be used by the sidebar navigation links (`href="#your-page-id"`) and for smooth scrolling and active link highlighting.

3. CSS Styling

The primary styling comes from Bootstrap 5, customized further by `css/style.css`. Classes like `mb-5` (margin-bottom), `doc-section`, `h3`, `fw-bold`, `border-bottom`, `pb-2`, `mb-4` are used extensively to maintain a consistent look and feel across all documentation pages.

Feel free to inspect elements using your browser's developer tools to understand how styles are applied.
